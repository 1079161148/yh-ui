# AI SDK Official Integration

`@yh-ui/ai-sdk` is the official AI capability integration package from YH-UI, seamlessly integrating **Vercel AI SDK** and **LangChain** for out-of-the-box usage.

## Features

- 🌐 **Multi-Provider Support** - OpenAI / Anthropic / Google / DeepSeek / Tongyi / Wenxin Yiyan, etc.
- 🔄 **Streaming Output** - Complete streaming text/object generation support
- 🛠️ **Function Calling** - Easily define and execute tool functions
- 🔗 **LangChain Integration** - Deep integration with LangChain.js ecosystem
- 🖥️ **Native Vue 3** - Full Composition API support
- 💾 **Conversation Management** - Built-in history persistence

## Installation

```bash
# Core packages
pnpm add @yh-ui/ai-sdk ai @langchain/core

# Install the provider package(s) you need
pnpm add @ai-sdk/openai    # OpenAI (GPT-4o, GPT-4, etc.)
# or
pnpm add @ai-sdk/anthropic  # Anthropic (Claude)
# or
pnpm add @ai-sdk/google    # Google Gemini
# or
pnpm add @ai-sdk/deepseek  # DeepSeek
# or
pnpm add @ai-sdk/ollama    # Ollama (local models)
```

::: tip
The `ai` package does not include model providers. You need to install the corresponding `@ai-sdk/*` package to use different models.
:::

::: warning Code Examples Note
The code examples in this documentation (especially those using `createOpenAI` and other providers) are **complete runnable examples**. You need to:

1. Install the corresponding `@ai-sdk/*` provider package
2. Configure a valid API Key (via environment variable or directly passed)

**The interactive buttons in the preview area** use mock data for UI demonstration only and do not call real APIs.
:::

#### Create HTTP Handler

Quickly create an MCP HTTP endpoint in Next.js or Express:

```typescript
// app/api/mcp/route.ts (Next.js App Router)
import { createMCPServerHTTPHandler } from '@yh-ui/ai-sdk'

export const GET = createMCPServerHTTPHandler({
  name: 'my-ai-server',
  version: '1.0.0',
  tools: [
    {
      name: 'search',
      description: 'Search for information',
      inputSchema: { type: 'object', properties: { query: { type: 'string' } } },
      execute: async ({ query }) => [{ type: 'text', text: `Results for: ${query}` }]
    }
  ]
})

export const POST = GET
```

### Multi-Server Management

Use `useMCPTools` to manage multiple MCP Servers:

```typescript
import { useMCPTools } from '@yh-ui/ai-sdk'

const { allTools, serverStates, callTool } = useMCPTools({
  servers: [
    { serverUrl: 'http://localhost:3001/mcp' },
    { command: 'npx', args: ['-y', '@modelcontextprotocol/server-filesystem', './data'] }
  ],
  autoConnect: true
})

// Call tool - automatically routes to correct server
const result = await callTool('tool_name', { arg: 'value' })
```

::: warning Notes

1. **MCP Server needs to implement JSON-RPC protocol**, following MCP spec version 2024-11-05
2. **Stdio mode only works in Node.js environment**, browser can only use HTTP mode
3. **HTTP mode requires CORS configuration**, ensure cross-origin requests are handled properly
4. **Tool calls are async**, recommend adding loading state to prompt users
   :::
